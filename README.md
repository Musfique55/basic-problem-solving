# TypeScript Quick Reference

## 1. Difference Between `interface` and `type`

**ইন্টারফেজ ওবজেক্ট এবং ক্লাস কন্সট্রাক্টরের সাথে ভালোভাবে খাপ খাওয়ায়। অন্যদিকে টাইপ কঠিন থেকে সহজ সব টাইপ সুন্দরভাবে ডেসক্রাইব করতে পারে।**

**Interface:**  
- মূলত অবজেক্টের গঠন বর্ণনা করতে ব্যবহৃত হয়।  
- প্রোপার্টি এবং মেথডগুলির একটি সেট সংজ্ঞায়িত করা হয়।  
- এটি একটি চুক্তি বা ব্লুপ্রিন্ট হিসাবে কাজ করে যা ক্লাস বা অবজেক্টকে অবশ্যই মেনে চলতে হবে।  
- উদাহরণ:
```ts
interface User {
  name: string;
  age: number;
}
```
**Type:**

- আরও বহুমুখী।

- এটি অবজেক্টের গঠন, প্রিমিটিভ টাইপ (string, number, boolean), ইউনিয়ন টাইপ (string | number), ইন্টারসেকশন টাইপ (TypeA & TypeB) এবং ফাংশন সিগনেচারও বর্ণনা করতে পারে।

- টাইপ ব্যবহার করে নতুন টাইপ তৈরি করা যায় যা বিদ্যমান টাইপগুলির সংমিশ্রণ বা পরিবর্তন।

- একটি প্রধান পার্থক্য হলো, ইন্টারফেসকে implements ব্যবহার করে ক্লাস দ্বারা ইমপ্লিমেন্ট করা যায়, যা টাইপসের ক্ষেত্রে সম্ভব নয়।

**সংক্ষেপে:**

- অবজেক্টের গঠন বা ক্লাসের চুক্তি → Interface

- জটিল বা বহুমুখী টাইপ → Type

## 2. Difference between  `any`, `unknown`, and `never` types in TypeScript :

**এই অংশে টাইপস্ক্রিপ্টের তিনটি গুরুত্বপূর্ণ টাইপের মধ্যে পার্থক্য দেখানো হয়েছে।**

---

## **1. any**
- যখন ব্যবহার করা হয় তখন যা খুশি তা করা যায়।  
- টাইপস্ক্রিপ্ট কোনো এরোর বা ওয়ার্নিং দিবে না।  

```ts
let value: any = 5;
value = "hello"; // allowed
value = true;    // allowed
```
## **2. unknown**

- কোনো অপারেশন করার আগে আলাদাভাবে ডাটা টাইপ চেক দিয়ে কাজ করতে হয়।
```ts
let value: unknown;

value = "hello";

if (typeof value === "string") {
  console.log(value.length); // safe
}
```
## **3. never**

- বোঝানো হয় যে এই ফাংশন বা ভেরিয়েবল কখনো কিছু রিটার্ন করবে না।

- সাধারণত error throw বা infinite loop এর জন্য ব্যবহার হয়।
```ts
function error(): never {
  throw new Error("Something went wrong");
}
```
